### YOLO 训练与使用完整步骤

**1. 环境准备**
- 确保你已经安装并激活了相应的 conda 环境（当前使用的环境是 `yolo`）：
  ```bash
  conda activate yolo
  ```
- 如果还未安装依赖，进入 `Model Traning` 目录并使用 requirements 安装（若适用）：
  ```bash
  cd "Model Traning"
  pip install -r requirements.txt
  ```

**2. 数据准备与划分**
- 将标注好的原始数据集存放到 `dataset_org` 目录下：
  - 图片放在 `dataset_org/images/`
  - YOLO格式标签代码存放在 `dataset_org/labels/`
- 运行数据划分脚本，程序会自动将数据集拆分为训练集(train)和验证集(val)，并重新组织文件：
  ```bash
  python data_split.py
  ```

**3. 配置文件修改**
- 打开 `data.yaml` 文件，确认数据集路径以及类别信息是否正确：
  - `path`: 基本数据路径
  - `train`: 训练集的相对路径或绝对路径
  - `val`: 验证集的相对路径或绝对路径
  - `nc`: 类别总数
  - `names`: 对应的类别名称列表

**4. 模型训练**
- 运行训练脚本，开始训练模型并生成权重文件。脚本内部应已配置好利用当前目录下的 `yolo11n.pt` 作为预训练权重：
  ```bash
  python train.py
  ```
- 训练完成后，模型权重（`best.pt` 等）会被保存在 `runs/detect/train/weights/` （或其他类似 run 标识）目录下。

**5. 模型评估与测试**
- 运行测试脚本，用于在验证集或测试集上评估最新的模型精度：
  ```bash
  python test.py
  ```
- *（注：测试脚本通常需要你在代码内部将权重路径指定为刚训练出的 `best.pt`）*

**6. 模型推理 (预测)**
- 使用训练好的模型对单张图像或视频进行目标检测：
  ```bash
  python predict.py
  ```
- 预测的结果通常会自动保存在 `runs/detect/predict/` 目录中。

**7. 模型导出 **
- 如果需要将模型部署到其他平台（如同ONNX, TensorRT等），运行导出脚本：
  ```bash
  python export.py
  ```
**8.模型转换**
- 在终端中启用WSL
-输入命令docker start -ai k230 ,启用k230容器
-准备校准数据集(可用训练数据集中的test集),copy到容器中
